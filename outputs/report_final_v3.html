Competition Overview

Predict the transcriptomic consequence of knocking down a gene — using only the identity of that gene, with no cell-level observations at test time. This is a true zero-shot generalisation challenge.

pert_signal = MLP(h_base[pert_idx])

Instead of predicting all 5,127 DE values directly, the model predicts 64 "gene program scores" (W), then decodes via a fixed SVD basis matrix (H) derived from the training data.

Key Innovations in GNN v2

01
Gene Programs (SVD)
Compresses 5127D output to 64D via fixed SVD basis. Reduces effective output space by 80x, drastically improving generalisation. 98% variance retained.
02
Curriculum Loss
Huber loss (robust to outliers) for epochs 1–20, then MAE (matches evaluation metric exactly) for epochs 21+. Stabilises early training, then optimises target metric directly.
03
MCDropout TTA
5 folds × 30 stochastic forward passes = 150 predictions averaged per test gene. Dropout kept active at inference time — calibrated Bayesian ensemble at no extra training cost.
04
DropEdge Regularisation
10% of graph edges randomly dropped per batch during training. Prevents over-reliance on any single edge, forces more robust neighbourhood aggregation.
05
Gated Decoder
A sigmoid gate computed from the perturbed gene embedding scales the decoder output. Allows the model to suppress or amplify different gene program channels per perturbation.
06
Raw Feature Residual
Direct skip connection from the 119D raw node features to the output. Ensures that basic expression statistics are always available to the decoder regardless of GNN depth.

6.2 TransformerConv — Attention Formula

Graph Attention Networks (GATv1) use an additive attention mechanism that concatenates node features and passes them through a learned vector. TransformerConv uses scaled dot-product attention — the same mechanism behind the Transformer architecture — but applied to graph message passing. This produces richer, better-calibrated attention scores.

Attention Formula Comparison
Standard GAT: α_ij = softmax( LeakyReLU( aT · [Wh_i ‖ Wh_j] ) ) # single learned vector a — limited expressivity TransformerConv: α_ij = softmax( (Q_i · K_j + W_e·e_ij) / √d_k ) # separate Q,K projections + edge feature term Where: Q_i = W_Q · h_i (query: what does node i want?) K_j = W_K · h_j (key: what does neighbour j offer?) e_ij = EdgeEncoder(w_ij, type_ij) ← edge features modulate α β = learnable scalar ∈ [0,1], data-driven skip Output aggregation: h_out[i] = β·h_in[i] + (1-β)·Σ_j α_ij · (V_j + W_e·e_ij)
Why TransformerConv is Better Here
  • Richer gradient signal: Dot-product attention is fully differentiable with sharper gradients than LeakyReLU-based additive attention — learns to focus on relevant edges faster.
  • Edge feature integration: The W_e·e_ij term directly incorporates our 32D edge encoding into both the attention score and the value aggregation — GAT only uses edge weights as scalar multipliers.
  • Learnable β skip: The model learns how much to retain vs. aggregate per layer, adapting depth of propagation to the data rather than using a fixed residual weight.
  • Separable Q/K projections: Different projection matrices for queries and keys let the model ask "what neighbourhood information does this gene need?" separately from "what information does this gene provide to others?"
6.3 Transcription Factor Regulatory Edges

Transcription factors (TFs) are master regulators of gene expression. When a TF is knocked out, its target genes lose their regulatory signal — causing a cascade of differential expression changes. This causal biology is not captured by undirected co-expression or PPI edges, which only encode correlation. Directed TF→target edges encode the actual regulatory mechanism.

Why TF edges matter for this competition

Problem with v2 edges
Co-expression edges are undirected correlations — if TF A activates gene B, A and B will be correlated, but so will many other gene pairs. The model cannot distinguish "A causes B" from "A and B share an upstream regulator". When A is knocked out, v2 cannot reason about the causal chain.
How v3 TF edges fix this
Directed TF→target edges encode "A directly regulates B". The TransformerConv model learns to treat these edges as a separate signal channel (via EdgeEncoder type embeddings). When A is perturbed, the model follows the directed edges to propagate the effect through the regulatory network.

6.4 EdgeEncoder Architecture

The EdgeEncoder converts raw edge attributes (a scalar score and an integer type) into a rich 32D vector that participates in both the attention score computation and the value aggregation. It is computed once per forward pass after DropEdge, then shared across all perturbation samples in the batch — maximally efficient.

edge_attr_raw: (E, 2)  = [score ∈ [0,1], type ∈ {0,1,2}]
↓ split
TypeEmbedding(3 → 16D)
Embedding table: 3 × 16
WeightProjection(1 → 16D)
Linear layer on scalar score
↓ concat → (E, 32D)
Linear(32 → 32) → ReLU → Dropout(0.1)
↓
e_ij: (E, 32D)  ← edge features ready for TransformerConv
# EdgeEncoder forward pass (PyTorch pseudocode) class EdgeEncoder(nn.Module): def __init__(self): self.type_emb = nn.Embedding(3, 16) self.weight_proj = nn.Linear(1, 16) self.out = nn.Sequential( nn.Linear(32, 32), nn.ReLU(), nn.Dropout(0.1) ) def forward(self, edge_attr): # edge_attr: (E, 2) — [score, type] w = self.weight_proj(edge_attr[:, 0:1]) # (E,16) t = self.type_emb(edge_attr[:, 1].long()) # (E,16) return self.out(torch.cat([w, t], dim=-1)) # (E,32)
Efficiency note: EdgeEncoder is called once per mini-batch (after DropEdge applies stochastic edge masking). The resulting 32D tensor is then broadcast across all (N_pert_per_batch) perturbation forward passes. This keeps the overhead to O(E × 32) per batch, independent of how many perturbations are in the batch.
6.5 GlobalContextLayer — O(N) Long-Range Interactions

Standard GNN message passing is local — information propagates one hop per layer. With L=3 layers, a gene can only receive information from genes within 3 graph hops. Many TF regulatory effects span longer paths, especially for master regulators with broad influence (e.g., TP53, MYC). The GlobalContextLayer provides an O(N) approximation to full self-attention that allows every gene to be influenced by a summary of the entire graph state.

# GlobalContextLayer forward pass def forward(self, h, batch=None): # Step 1: Compute graph-level summary vector h_global = global_mean_pool(h, batch) # (B, H) g = self.ctx_proj(h_global) # (B, H) — learned projection # Step 2: Broadcast to node level g_expanded = g[batch] # (N, H) # Step 3: Compute per-node gate gate = torch.sigmoid( self.gate_proj(torch.cat([h, g_expanded], dim=-1)) ) # (N, H) ∈ [0,1] # Step 4: Gated fusion + residual h_ctx = gate * g_expanded # (N, H) — context contribution return self.norm(h + self.dropout(h_ctx)) # (N, H) — residual + LayerNorm
Why O(N) and not O(N²)?

Full self-attention computes pairwise scores between all N=5,155 gene nodes — a 5,155² ≈ 26.6M attention matrix. The GlobalContextLayer instead computes a single graph-level summary via mean pooling (O(N)), then gates each node's contribution from that summary. This is analogous to the "register tokens" in Vision Transformers or the global node trick in graph transformers.

Biological Motivation

Master TFs like TP53 and MYC regulate hundreds of genes across the genome. Their effect is not local to a graph neighbourhood — it is global. When TP53 is knocked out, the global expression state changes in a way that affects every gene. The GlobalContextLayer captures this: the summary vector h_global encodes the "expression state" and gates each gene's response to it.

6.6 Why GNN v3 Outperforms v2
Biological Reasoning
  • TF edges encode causal regulatory biology — CRISPR knockouts are causal interventions, not correlational. Causal edges should improve prediction of downstream effects.
  • Activating vs. repressing TF edges carry directional sign information that co-expression edges cannot represent (correlation is symmetric).
  • The GlobalContextLayer allows master regulator effects (TP53, MYC, CTCF) to propagate globally without requiring a deep chain of local hops.
Architectural Reasoning
  • TransformerConv dot-product attention has higher expressivity than additive GAT — learns richer query/key matchings for which gene pairs are most relevant per perturbation.
  • The EdgeEncoder teaches the model to weight TF edges differently from PPI, preventing the high-volume TF edges from overwhelming the PPI signal.
  • Learnable β residual scalar allows the model to adaptively control message passing depth — shallower where local structure suffices, deeper where long-range context matters.

Ensemble Strategy

Models with low prediction correlation are complementary for ensembling. GNN v2 captures graph topology while KNN and Ridge capture feature similarity — blending them reduces the unique errors of each approach.

Optimal Blend Formula
pred_ensemble = w1 · GNN_v2
+ w2 · KNN
+ w3 · Ridge
+ w4 · GNN_v3

Weights w1..w4 are optimised by minimising OOF WMAE using Optuna Bayesian optimisation or a simple grid search. Constraint: Σw_i = 1, w_i ≥ 0. Out-of-fold predictions from 5-fold CV are used as the target for weight fitting.
