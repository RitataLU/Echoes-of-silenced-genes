Background

What Problem Are We Solving?

CRISPR knocks out a gene. We need to predict how every other gene in the cell responds — without ever seeing that gene knocked out before.

📥 Input — one gene name

e.g. "knock out TP53"
That's all we know. No cell data, no experiment.

📤 Output — 5,127 numbers

DE[gene] = mean(knocked-out cells) − mean(normal cells)
For every measurable gene in the cell.

⚡ The hard part — zero-shot

120 test genes were never knocked out during training.
The model must generalise from the 80 training knockouts.

DE = mean(perturbed) − mean(non-targeting control)

Method 1

KNN — K-Nearest Neighbours

The simplest idea: if two genes look similar, they probably respond similarly to perturbation.

How It Works (3 steps)

1
Represent each gene as a vector
Use the 119D node features (expression + co-exp + GO) to describe each gene numerically.
2
Find the 5 most similar training genes
Compute cosine/Euclidean distance in 119D space. Return top-5 neighbours that were actually knocked out in training.
3
Average their DE profiles
Prediction = mean of the 5 neighbours' known DE vectors (5127D each).

✓ Strengths
  • No training required
  • Interpretable — you can see which genes were used
  • Fast (seconds to run)
  • Strong when similar genes exist
✗ Limitations
  • Averages out subtle differences
  • Fails for unique genes with no close neighbours
  • Ignores indirect regulatory paths
  • Can't model non-linear interactions

⬡ Stage 1 — Context Encoder

Run 2 layers of GAT (Graph Attention) over all 5155 genes. Each gene gathers information from its neighbours. Result: h_base = "what does this gene look like in a normal cell, given its network context?"

⚡ Stage 2 — Perturbation Propagation

Take the knocked-out gene's embedding. Build a perturbation signal. Broadcast it to ALL nodes, then run GAT again. Result: h_final = "how does each gene change after the knockdown signal propagates through the network?"

🧬 Gene Programs (SVD)

Instead of predicting 5127 numbers directly, predict 64 program scores. Each program = a group of genes that go up/down together. Decode: DE = W_scores @ H_matrix

5 Key Innovations in v2

①
Gene Programs

Output 64D instead of 5127D. Reduces task complexity, improves generalisation.

②
Curriculum Loss

Epoch 1–20: Huber (tolerates outliers). Epoch 21+: Weighted MAE (matches competition metric).

③
MCDropout TTA

At inference: keep dropout ON, average 30 passes. Free ensemble from one model.

④
DropEdge

Randomly remove 10% edges per batch. Forces the model to not over-rely on any single connection.

Result

OOF WMAE = 0.1167  ·  −7.9% vs baseline  ·  Best model until GNN v3.

Upgrade 1 — TransformerConv Attention

GAT (v2): Additive attention

α = softmax( LeakyReLU( a · [Wh_i ‖ Wh_j] ) ) -- learns a fixed "scoring function" for each pair -- can't use edge features directly

TransformerConv (v3): Dot-product attention

α = softmax( (Q_i · K_j + W_e · e_ij) / √d ) -- Q_i = query (what am I looking for?) -- K_j = key (what does neighbour offer?) -- e_ij = EdgeEncoder output ← edge features! -- β = learnable skip: h = β·h_in + (1-β)·agg

Why graph-based methods dominate

Graph Methods (GNN)

Use indirect regulatory paths. Gene A → Gene B → Gene C. Knockdown of A affects C even without a direct edge. LGBM/MLP treat each gene pair independently — they miss this cascade.

KNN (feature similarity)

Averages across similar training genes. Works well when the test gene has close neighbours in feature space. Fails for unique genes or when indirect effects dominate.

LGBM / MLP / CPA

Model each (pert, target_gene) pair independently. No network topology awareness. Good at interpolating within known training distribution, bad at zero-shot extrapolation.
